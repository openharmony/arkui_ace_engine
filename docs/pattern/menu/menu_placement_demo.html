📋 Placement 避让顺序

💡 提示：避让机制的核心思想是"同方向优先，必要时反向"：
  • 如果放下方不够，尝试放上方
  • 如果放左边不够，尝试放右边
  • 如果空间还不够，会尝试对角位置 - 见 GetAdjustPosition() (menu_layout_algorithm.cpp:3140)
  • 最后如果都不行，会尝试填满屏幕 - 见 CalcVerticalPosition() (menu_layout_algorithm.cpp:2830-2845)

💻 核心代码逻辑

1. 位置计算入口

// menu_layout_algorithm.cpp:2084-2133
OffsetF MenuLayoutAlgorithm::UpdateMenuPosition(...)
{
    auto avoidanceMode = menuProp->GetSelectAvoidanceMode()
        .value_or(AvoidanceMode::COVER_TARGET);
    
    if (useLastPosition) {
        // 保持上次位置
        menuPosition = lastPosition_.value();
    } else if (menuPattern->IsSelectMenu() && 
               avoidanceMode == AvoidanceMode::AVOID_AROUND_TARGET) {
        // Select菜单避让模式
        menuPosition = SelectLayoutAvoidAlgorithm(...);
    } else {
        // 普通菜单布局
        menuPosition = MenuLayoutAvoidAlgorithm(...);
    }
}

2. 垂直布局计算

// menu_layout_algorithm.cpp:2797-2845
float MenuLayoutAlgorithm::VerticalLayout(
    const SizeF& size, float position, bool isContextMenu)
{
    placement_ = Placement::BOTTOM;
    
    // 1. 优先放下面
    if (GreatOrEqual(bottomSpace_, size.Height())) {
        return position + margin_;
    }
    
    // 2. 放下面不够，尝试放上面
    if (GreatOrEqual(topSpace_, size.Height())) {
        placement_ = Placement::TOP;
        return topSpace_ - size.Height() + margin_;
    }
    
    // 3. 都放不下，填满屏幕底部
    if (LessNotEqual(size.Height(), wrapperHeight)) {
        return wrapperHeight - size.Height();
    }
    
    // 4. 完全放不下，贴顶
    return 0.0f;
}

4. 屏幕适配检查

// menu_layout_algorithm.cpp:3037-3055
OffsetF MenuLayoutAlgorithm::FitToScreen(
    const OffsetF& position, const SizeF& childSize, bool didNeedArrow)
{
    // 添加用户偏移
    OffsetF afterOffsetPosition;
    if (NearEqual(positionOffset_, OffsetF(0.0f, 0.0f))) {
        afterOffsetPosition = AddTargetSpace(originPosition);
    } else {
        afterOffsetPosition = AddOffset(originPosition);
    }
    
    // 检查是否在安全区域内
    if (!CheckPosition(afterOffsetPosition, childSize) || flag_) {
        return OffsetF(0.0f, 0.0f);  // 返回0表示不可用
    }
    
    return afterOffsetPosition;
}

5. 对角位置避让 (GetAdjustPosition)

// menu_layout_algorithm.cpp:3140-3162
// 当所有主placement都失败后，尝试按step跳过对齐方式来尝试对角位置
OffsetF MenuLayoutAlgorithm::GetAdjustPosition(
    std::vector<Placement>& currentPlacementStates, size_t step,
    const SizeF& childSize, const OffsetF& topPosition, const OffsetF& bottomPosition)
{
    for (size_t i = 0; i < len;) {
        placement_ = currentPlacementStates[i];
        if (placement_ == Placement::NONE) {
            break;  // 遇到NONE停止
        }
        childPosition = GetPositionWithPlacement(childSize, topPosition, bottomPosition);
        position = AdjustPosition(childPosition, ...);
        if (!NearEqual(position, OffsetF(0.0f, 0.0f))) {
            break;  // 找到可用位置
        }
        i += step;  // 跳过step个placement，尝试对角位置
    }
    return position;
}

6. 填满屏幕 (CalcVerticalPosition)

// menu_layout_algorithm.cpp:2830-2845
float MenuLayoutAlgorithm::CalcVerticalPosition(const SizeF& size)
{
    // 1. 尝试放上面
    if (GreatOrEqual(topSpace_, size.Height())) {
        placement_ = Placement::TOP;
        return topSpace_ - size.Height() + margin_;
    }
    
    // 2. 填满屏幕底部
    if (LessNotEqual(size.Height(), wrapperHeight)) {
        return wrapperHeight - size.Height();
    }
    
    // 3. 完全放不下，贴顶
    return 0.0f;
}

🛡️ 避让规则详解

参数 作用 来源
wrapperRect_ 考虑安全区域后的可用区域 由 safeAreaInsets 计算
targetSecurity_ 目标组件周围的保留间距 主题中获取 (默认8vp)
paddingStart_/paddingEnd_ 菜单左右内边距 主题配置
topSecurity/bottomSecurity 上下安全区域边界 由设备屏幕决定

避让模式对比

模式 枚举值 行为描述 代码位置
覆盖目标 COVER_TARGET 菜单直接显示在目标位置，可能覆盖目标组件 menu_layout_algorithm.cpp:2098
环绕目标 AVOID_AROUND_TARGET 菜单尽量显示在目标周围，不覆盖目标 menu_layout_algorithm.cpp:2106

📌 关键点：
  • 系统会自动计算安全区域（状态栏、导航栏、挖孔屏等）
  • 在屏幕边缘时，菜单会自动调整位置避免超出屏幕
  • 设置了PositionOffset会叠加在计算结果上
  • Preview菜单有特殊的布局逻辑处理动画
